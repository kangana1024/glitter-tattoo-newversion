# Specification: Rebuild glitter-tattoo.com as Modern Next.js SSG Site

## Overview

Rebuild the existing website [glitter-tattoo.com](https://www.glitter-tattoo.com) from scratch as a modern, SEO-optimized static site using Next.js (latest stable) with Static Site Generation (SSG). The new site will support 3 languages (Thai, English, and one additional locale), feature a completely modernized UI with a vibrant multi-color palette inspired by the brand logo, scrape and locally store all content/images from the existing site, implement a new URL structure while preserving legacy URL resolution for SEO continuity, and deploy as a fully static export. This is a **greenfield build** — no existing codebase to work from.

## Workflow Type

**Type**: feature

**Rationale**: This is a full greenfield project requiring creation of an entire Next.js application from scratch, including project scaffolding, content scraping infrastructure, i18n routing, UI component library, and SEO optimization. While the requirements doc marks it as "documentation", the actual work is a large-scale feature build.

## Task Scope

### Services Involved
- **next-app** (primary) - The Next.js application: pages, components, layouts, i18n, SEO metadata
- **scraper** (tooling) - Node.js scripts to crawl the existing site, extract content/images, and generate local data files
- **static-export** (deployment) - The `/out` directory generated by `next build` for static hosting

### This Task Will:
- [ ] Initialize a Next.js project with App Router, Tailwind CSS v4, and next-intl
- [ ] Configure `next.config.ts` for static export (`output: 'export'`) with next-intl plugin
- [ ] Set up i18n routing with `app/[locale]/` structure for 3 locales (en, th, zh)
- [ ] Build content scraping scripts to extract all text, images, and page data from the existing site
- [ ] Create a URL mapping system (old URLs → new URLs) with client-side redirect support
- [ ] Design and implement a modern, vibrant UI with component library
- [ ] Implement full SEO support: metadata, Open Graph, structured data, sitemap, robots.txt
- [ ] Store all scraped content locally as JSON data files and downloaded images
- [ ] Build all pages as statically generated routes with `generateStaticParams()`

### Out of Scope:
- Server-side rendering (SSR) — this is SSG/static export only
- CMS integration — all content is local JSON files
- User authentication or dynamic user features
- E-commerce/payment processing
- Server-side redirects (not possible with static export — using client-side redirects instead)
- Email/contact form backend (frontend form only; backend integration deferred)

## Patterns to Follow

### 1. Next.js Static Export Configuration

```typescript
// next.config.ts
import createNextIntlPlugin from 'next-intl/plugin';

const withNextIntl = createNextIntlPlugin('./lib/i18n-request.ts');

const nextConfig = {
  output: 'export',
  images: {
    unoptimized: true,
  },
  trailingSlash: true,
};

export default withNextIntl(nextConfig);
```

**Key Points:**
- `output: 'export'` produces static HTML to `/out`
- `images.unoptimized: true` required — no server for image optimization
- `trailingSlash: true` for clean static URLs
- Wrap config with `createNextIntlPlugin()`

### 2. i18n Setup (next-intl without middleware)

```typescript
// lib/i18n.ts
export const locales = ['en', 'th', 'zh'] as const;
export const defaultLocale = 'th' as const;
export type Locale = (typeof locales)[number];
```

```typescript
// lib/i18n-request.ts
import { getRequestConfig } from 'next-intl/server';
import { locales, type Locale } from './i18n';

export default getRequestConfig(async ({ requestLocale }) => {
  let locale = await requestLocale;
  if (!locale || !locales.includes(locale as Locale)) {
    locale = 'th';
  }

  return {
    locale,
    messages: (await import(`../messages/${locale}.json`)).default,
  };
});
```

```typescript
// app/[locale]/layout.tsx
import { setRequestLocale } from 'next-intl/server';
import { NextIntlClientProvider } from 'next-intl';
import { getMessages } from 'next-intl/server';
import { notFound } from 'next/navigation';
import { locales } from '@/lib/i18n';

export function generateStaticParams() {
  return locales.map((locale) => ({ locale }));
}

export default async function LocaleLayout({
  children,
  params,
}: {
  children: React.ReactNode;
  params: Promise<{ locale: string }>;
}) {
  const { locale } = await params;
  if (!locales.includes(locale as any)) notFound();
  setRequestLocale(locale);

  const messages = await getMessages();

  return (
    <NextIntlClientProvider locale={locale} messages={messages}>
      {children}
    </NextIntlClientProvider>
  );
}
```

**Key Points:**
- Every page/layout MUST call `setRequestLocale(locale)` for static generation
- Every page MUST export `generateStaticParams()` returning all locales
- Root `/` page does client-side redirect to `/${defaultLocale}`
- No middleware — static export does not support it

### 4. Client-Side Legacy URL Redirect Pattern

```typescript
// app/[locale]/[...rest]/page.tsx
import { redirect, notFound } from 'next/navigation';
import { redirectMap } from '@/lib/redirect-map';
import { locales } from '@/lib/i18n';

export function generateStaticParams() {
  // Generate params for all known legacy URLs across all locales
  return Object.keys(redirectMap).flatMap((oldPath) =>
    locales.map((locale) => ({
      locale,
      rest: oldPath.split('/').filter(Boolean),
    }))
  );
}

export default async function CatchAllPage({
  params,
}: {
  params: Promise<{ locale: string; rest: string[] }>;
}) {
  const { locale, rest } = await params;
  const oldPath = '/' + rest.join('/');
  const newPath = redirectMap[oldPath];

  if (newPath) {
    redirect(`/${locale}${newPath}`);
  }

  // Show 404 for truly unknown paths
  notFound();
}
```

**Key Points:**
- Static export cannot use server redirects — must use catch-all route
- `generateStaticParams()` pre-renders pages for all known legacy URLs
- Each legacy URL gets a static HTML page that redirects via Next.js `redirect()`
- Also generate `<meta http-equiv="refresh">` as fallback

### 5. Content Scraping Pattern

```typescript
// scripts/scrape-content.ts
import * as cheerio from 'cheerio';

interface PageContent {
  url: string;
  title: string;
  description: string;
  headings: string[];
  bodyHtml: string;
  bodyText: string;
  images: { src: string; alt: string }[];
  links: { href: string; text: string }[];
  locale: string;
}

async function scrapePage(url: string): Promise<PageContent> {
  const response = await fetch(url);
  const html = await response.text();
  const $ = cheerio.load(html);

  return {
    url,
    title: $('title').text(),
    description: $('meta[name="description"]').attr('content') || '',
    headings: $('h1, h2, h3').map((_, el) => $(el).text()).get(),
    bodyHtml: $('body').html() || '',
    bodyText: $('body').text().trim(),
    images: $('img').map((_, el) => ({
      src: $(el).attr('src') || '',
      alt: $(el).attr('alt') || '',
    })).get(),
    links: $('a[href]').map((_, el) => ({
      href: $(el).attr('href') || '',
      text: $(el).text().trim(),
    })).get(),
    locale: detectLocale(url),
  };
}
```

**Key Points:**
- Use Cheerio for static HTML parsing (lightweight, fast)
- Fall back to Puppeteer only for JS-rendered content
- Store raw scraped data in `content/scraped/`
- Process into structured JSON in `content/pages/`
- Download all images to `public/images/scraped/`

   - Description: Support 3 languages (Thai, English, Chinese) with sub-path routing (`/th/`, `/en/`, `/zh/`)
   - Acceptance: Every page accessible in all 3 locales; language switcher works; Thai is default locale

4. **Modern UI Redesign**
   - Description: Design a modern, responsive UI with a vibrant multi-color palette inspired by the glitter-tattoo brand logo
   - Acceptance: All pages display correctly on mobile/tablet/desktop; color palette matches brand identity; design is modern and professional

5. **Legacy URL Support**
   - Description: All old URLs from the existing site must resolve to the corresponding new page via client-side redirects
   - Acceptance: Accessing any old URL redirects to the correct new page; redirect map is comprehensive

6. **SEO Optimization**
   - Description: Full SEO support including meta tags, Open Graph, structured data (JSON-LD), sitemap.xml, and robots.txt
   - Acceptance: Lighthouse SEO score > 90; all pages have unique titles/descriptions; sitemap includes all pages in all locales

### Edge Cases

1. **Missing content on existing site** — Some pages may be broken or return 404; scraper must handle gracefully and log warnings
2. **Non-Thai/English content** — Existing site may have mixed-language content; scraper must detect and categorize
3. **Relative image URLs** — Scraper must resolve relative URLs to absolute before downloading
4. **Duplicate images** — Same image used across pages; download once, reference everywhere
5. **Very large images** — Optimize/compress images during scraping to keep static bundle size manageable
6. **JavaScript-rendered content** — Some content may require Puppeteer if Cheerio can't access it
7. **Root URL without locale** — `/` must redirect to `/th/` (default locale) via client-side redirect
8. **Unknown legacy URLs** — URLs not in the redirect map should show a proper 404 page

## Implementation Notes

### DO
- Use `generateStaticParams()` in EVERY dynamic route page and layout
- Call `setRequestLocale(locale)` in EVERY Server Component page/layout
- Use `unoptimized: true` for all Next.js `<Image>` components
- Use Thai-compatible Google Fonts (Kanit for headings, Sarabun for body)
- Keep all content as local JSON files — no external API calls at runtime
- Use semantic HTML elements for SEO (`<article>`, `<nav>`, `<main>`, `<section>`)
- Generate `<meta http-equiv="refresh">` tags as redirect fallback
- Implement proper `<html lang="">` attribute per locale
- Use `hreflang` alternate links for multi-language SEO
- Implement JSON-LD structured data for business info

### DON'T
- Don't use `proxy.ts` (formerly `middleware.ts` in v15) — not supported in static export
- Don't use Next.js Image Optimization API — use `unoptimized: true`
- Don't use `redirect()` in `next.config.ts` — not supported in static export
- Don't use `rewrite()` or `headers()` in config — not supported in static export
- Don't rely on server-side locale detection — use client-side redirect from root
- Don't use `'use cache'` directive — not needed for static export
- Don't import Puppeteer in the Next.js app — scraping scripts are separate Node.js scripts
- Don't hardcode text strings — all user-facing text goes through next-intl messages
